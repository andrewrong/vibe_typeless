# Typeless 监控与可用性设计

## 目标
- 减少不必要的日志输出
- 添加关键链路延迟监控
- 添加系统可用性监控

## 日志级别规范

### Swift (前端)
- **Error**: 系统错误、录音失败、API调用失败
- **Info**: 状态变化（开始录音、停止录音）、初始化完成
- **Debug**: 详细流程（音频buffer、session创建细节）

### Python (后端)
- **Error**: 异常、模型加载失败、服务错误
- **Info**: 请求处理、状态变化
- **Debug**: 详细处理流程、音频数据处理细节

## 监控指标

### 1. 链路延迟监控
| 指标 | 说明 | 告警阈值 |
|------|------|----------|
| audio_to_preview | 音频到预览延迟 | > 2s |
| stop_to_result | 停止到最终结果 | > 5s |
| session_create_time | Session创建时间 | > 1s |

### 2. 可用性监控
| 指标 | 说明 | 告警阈值 |
|------|------|----------|
| recording_success_rate | 录音成功率 | < 95% |
| asr_success_rate | ASR识别成功率 | < 90% |
| engine_start_success | 引擎启动成功率 | < 98% |

## 实施方案

1. **日志清理**
   - 将流程调试日志改为 debug 级别
   - 保留关键状态变化为 info
   - 错误保留为 error

2. **延迟监控**
   - 在关键节点添加时间戳记录
   - 计算并输出延迟指标

3. **可用性监控**
   - 记录成功/失败计数
   - 定期输出成功率统计
